# 变量

> [!note] 相关笔记
> 本文介绍 JavaScript 的变量声明与命名。前置知识见 [[JavaScript/01_Hello World!|Hello World]] 和 [[JavaScript/02_现代模式，"use strict"|use strict 严格模式]]。学习任务见 [[任务清单/2026-05-24|2026-05-24 任务清单]]。

## 变量声明

变量是数据的**"命名存储"**，在 JavaScript 中创建一个变量，需要用到 `let` 关键字。

下面的语句创建（也可以称作声明或定义）了一个名称为 `message` 的变量：

```javascript
let message;
```

可以通过赋值运算符 `=` 为变量添加一些数据：

```javascript
let message;

message = "hello"; // 将字符串 hello 保存在名为 message 的变量中
```

现在这个字符串已经保存到与该变量相关联的内存区域了，我们可以通过使用该变量名称访问它：

```javascript
let message;

message = "hello";

alert(message); // 显示变量内容
```

简洁一点，可以直接将变量定义与赋值合并为一行：

```javascript
let message = "hello"; // 定义变量并赋值

alert(message); // hello
```

也可以在一行中声明多个变量：

```javascript
let usr = "shiwu", age = 23, message = "hello";
```

> [!notice]
> 虽然可以一行声明多个变量，但为了**可读性**，请一行只声明一个变量。

> [!note] `var` 与 `let`
> 在较老的脚本中，你可能发现另一个关键字 `var`，而不是 `let`：
>
> ```javascript
> var message = "hello";
> ```
>
> `var` 关键字与 `let` 大体相同，也用来声明变量，但有些许不同，也有些"老派"。

## 现实类比

如果将变量想象成一个"数据"的盒子，盒子上有一个唯一的标注盒子名字的贴纸。这样我们更能轻松地掌握"变量"的概念。

例如，变量 `message` 可以想象成一个标有 `"message"` 的盒子，盒子的值为 `"hello"`：

![[JavaScript/assets/03_变量/file-20260524023708612.png]]

我们可以在盒子内放入任何值。并且，这个盒子的值想改变多少次就改变多少次：

```javascript
let message;

message = "hello";

message = "world";

alert(message);
```

当值改变的时候，之前的数据就从变量中删除了：

![[JavaScript/assets/03_变量/file-20260524023932388.png]]

我们还可以声明两个变量，然后将其中一个变量的数据拷贝到另一个变量：

```javascript
let hello = "hello world!";

let message;

// 将字符串 "hello world!" 从变量 hello 拷贝到 message
message = hello;

// 现在两个变量保存着相同的数据
alert(hello);
alert(message);
```

> [!warning] 变量不能重复声明
> 变量声明两次会触发 error，一个变量应该只被声明一次：
>
> ```javascript
> let message = "this";
> // 重复 let 会导致 error
> let message = "that"; // SyntaxError: 'message' has already been declared
> ```
>
> 因此，对同一个变量应该只声明一次，之后在不使用 `let` 的情况下对其进行引用。

## 变量命名

JavaScript 的变量命名有两个限制：

1. 变量名必须仅包含**字母、数字、符号 `$`、`_`**
2. 首字符必须**非数字**

如果命名包含多个单词，通常采用**驼峰式命名法**（camelCase）——单词一个接一个，除了第一个单词，其他的每个单词都以大写字母开头：`myVeryLongName`。

关于特殊字符的使用：

| 字符 | 是否可用 | 建议 |
|------|----------|------|
| `$` | 可用 | **不推荐** |
| `_` | 可用 | **不推荐** |
| 非英文字母 | 可用 | **不推荐** |

> [!notice] 区分大小写
> 变量 `apple` 和 `APPLE` 是两个不同的变量。

> [!important] 保留字
> 有一张[保留字列表](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Lexical_grammar#Keywords)，表中的保留字**无法用作变量命名**，因为它们被用于编程语言本身。

> [!warning] 未声明变量的隐患
> 在非严格模式下，直接给未声明的变量赋值，JavaScript 不会报错，而是悄悄把该变量挂到**全局对象**上：
>
> ```javascript
> x = 10;
> console.log(x);
> ```
>
> 全局变量会在整个程序生命周期存在，任何地方都能读写，**极难追踪 bug**。这也是为什么推荐始终使用 `"use strict"` ——严格模式下，该操作会直接抛出 `ReferenceError`。

## 常量

声明一个常数（不变）变量，可以使用 `const` 而非 `let`：

```javascript
const myBirthday = "18.04.1982";
```

使用 `const` 声明的变量称作**"常量"**。它们不能被修改，如果你尝试修改就会发现报错：

```javascript
const myBirthday = "18.04.1982";

myBirthday = "01.01.2001"; // 错误，不能对常量重新赋值
```

> [!tip] 何时使用 `const`
> 当程序员能确定这个变量永远不会改变的时候，就应该使用 `const` 来确保这种行为，并且清楚地向别人传递这一事实。

### 大写形式的常量

一个普遍的做法是将常量用作别名，以便记住那些在执行之前就已知的难以记住的值。

使用**大写字母和下划线**来命名这些常量。例如，以 web 十六进制格式为颜色声明常量：

```javascript
const COLOR_RED = "#F00";
const COLOR_GREEN = "#0F0";
const COLOR_BLUE = "#00F";
const COLOR_ORANGE = "#FF7F00";

// 当我们需要一个颜色
let color = COLOR_ORANGE;
alert(color);
```

这样做的好处：

- `COLOR_ORANGE` 比 `"#FF7F00"` 更容易记忆
- 比起 `COLOR_ORANGE` 而言，`"#FF7F00"` 更容易输错
- 阅读代码时，`COLOR_ORANGE` 比 `#FF7F00` 更易懂

## 正确命名变量

一些可以遵循的规则：

- 使用易读的命名，比如 `userName` 或者 `shoppingCart`
- 离诸如 `a`、`b`、`c` 这种缩写和短名称远一点，除非你真的知道你在干什么
- 变量名在能够准确描述变量的同时要足够简洁。不好的例子就是 `data` 和 `value`，这样的名称等于什么都没说。如果能够非常明显地从上下文知道数据和值所表达的含义，这样使用它们也是可以的
- 脑海中的术语要和团队保持一致。如果网站的访客称为"用户"，则我们采用相关的变量命名，比如 `currentUser` 或者 `newUser`，而不要使用 `currentVisitor` 或者 `newManInTown`
