# rsync 远程同步工具

> [!note] 相关笔记
> rsync 解决的本质问题是"如何高效地让两个地方的文件保持一致"。最笨的办法是每次全量复制，rsync 的聪明之处在于只传输差异部分——通过对比文件的校验和与时间戳，跳过已经相同的内容。

## 基本结构

```shell
rsync [选项] 来源 目标
```

来源和目标可以是本地路径，也可以是 `user@host:/path` 的远程路径（通过 SSH 连接）。

## 常用场景

### 本地同步

```shell
rsync -av /源目录 /目标目录/
```

### 推送到远程

```shell
rsync -av /本地/文件夹/ user@host:/远程/路径/
```

### 从远程拉取

```shell
rsync -av user@host:/远程/路径/ /本地/文件夹/
```

> [!notice]
> 不管是推送到远程还是从远程拉取，命令都在本地机器运行。rsync 通过 SSH 建立连接，本地必须能够 SSH 连接远程主机（密钥或密码均可）。

## 关键选项

| 选项 | 含义 |
|------|------|
| `-a` | **归档模式**：保留权限、时间戳、软链接等（几乎必用） |
| `-v` | 显示传输过程 |
| `-z` | 传输时压缩（网络慢时有用） |
| `-n` | **演习模式**，只显示会做什么，不真正执行 |
| `--delete` | 删除目标中源没有的文件（真正的"镜像同步"） |
| `--progress` | 显示每个文件的传输进度 |

> [!warning] 末尾斜杠的坑
> ```shell
> rsync -avz /src/ /dst/   # 同步 src 里的内容 → dst/
> rsync -avz /src /dst/    # 同步 src 目录本身 → dst/src/
> ```
> 源路径末尾是否有 `/` 决定行为，**务必留意**。
